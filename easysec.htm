01 — Project Overview

Financial analysts spend hours manually searching SEC filings for specific data points, often dealing with inconsistent formatting and unreliable AI-generated numbers. EasySEC solves this with a 5-stage query pipeline that classifies incoming queries, retrieves relevant filing passages through hybrid search, cross-validates numbers against XBRL structured data, and generates citation-backed responses.

The platform classifies queries into 5 types (NUMERIC_EXACT, NUMERIC_CALC, COMPARISON, TEXT_SEARCH, SYNTHESIS) and routes them through 3 execution paths (DETERMINISTIC for exact lookups, RETRIEVAL for semantic search, MIXED for hybrid). XBRL cross-validation reduces financial scaling errors 74x compared to plain text extraction, achieving a 0.11% error rate.

Built as a full-stack application with React frontend, FastAPI backend, PostgreSQL with pgvector for vector storage, OpenSearch for BM25, and FAISS for semantic indexing. The system includes a Knowledge Graph with entity/relation extraction, Semantic Diff using the Hungarian algorithm, and cross-encoder reranking for result quality.

03 — Architecture
Path A

Deterministic

For NUMERIC_EXACT queries. Bypasses semantic search entirely — routes directly to XBRL structured data for precise number extraction with zero ambiguity.

  • Direct XBRL data lookup
  • Zero-ambiguity numeric extraction
  • Sub-5ms classification routing
  • 0.11% scaling error rate
Path B

Retrieval

For TEXT_SEARCH and SYNTHESIS queries. Full hybrid search pipeline with BM25 + semantic fusion, cross-encoder reranking, and RAG response generation.

  • Hybrid BM25 + semantic search
  • Reciprocal Rank Fusion
  • Cross-encoder reranking
  • PageIndex navigation
Path C

Mixed

For NUMERIC_CALC and COMPARISON queries. Combines XBRL deterministic data with semantic context retrieval for calculations requiring both numbers and narrative.

  • XBRL + semantic fusion
  • Knowledge Graph traversal
  • Entity/relation extraction
  • Semantic Diff (Hungarian algo)
