01 — Project Overview

Panoptris is the AI reasoning layer that sits on top of existing cameras, sensors, and access control at remote critical infrastructure sites — solar farms, substations, cell towers, and pipelines. The product triages alerts, filters false alarms through cross-system correlation, and orchestrates the entire response chain: audio deterrents, notification routing, escalation, and drone dispatch when docks are present.

The positioning is camera-first, drone-ready. Lead with AI intelligence for the cameras already deployed; drone dispatch is the built-in upgrade path as dock adoption grows post-Part 108. Unlike point solutions, Panoptris reasons across five context streams simultaneously — camera feed, access control, zone history, organizational policy, and expected activity patterns — before classifying an event.

Architecture is a 4-tier decision model with structural separation enforced in code: the LLM is never in the safety-critical path. Multi-tenant from day one, full audit trails for SOC 2 / ISO 27001, and a research-backed intelligence pipeline verified end-to-end in v12.0 production hardening (March 2026).

Precursor — Sentrix

Panoptris began as Sentrix, an earlier exploration of drone-agnostic surveillance and inspection with an autonomous decision engine and graduated autonomy levels. The technical foundation — risk scoring, autonomous decisioning, drone-protocol abstraction — carried forward. The pivot was in positioning: from broad surveillance to a focused ICP of remote critical infrastructure operators where the false-alarm problem is most acute and the pain is most qualified.

03 — Architecture Decisions
Safety

4-Tier Decision Model

The LLM is never in the safety-critical path. Decisions flow through deterministic, policy-bounded, and human-confirmed tiers before reaching autonomous AI action.

  • T1: Deterministic rules
  • T2: Policy-bounded action
  • T3: LLM with guardrails
  • T4: Human confirmation
Memory

Postgres-Backed Agent Memory

No vector store, no separate memory service. Agent memory is structured Postgres queries assembled into context at decision time. Auditable, debuggable, fast.

  • Per-tenant data isolation
  • Zone & site history queries
  • Context assembly at runtime
  • SD-036 architectural decision
Drone Path

Camera-First, Drone-Ready

Core value works without drones. Drone dispatch is the upgrade path. Hardware abstraction supports DJI Cloud (MQTT) and MAVLink protocols.

  • MQTT for DJI Dock 2
  • MAVLink adapter
  • Don't replicate fleet management
  • Premium tier on dock adoption
