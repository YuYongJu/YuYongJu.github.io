01 — Project Overview

Aurelius watches what happens on your computer — focused windows, visible text, selected text, active URLs, open documents — and uses an LLM to elicit structured knowledge items from that activity. Entity resolution merges duplicates, Leiden clustering groups related items into themes, and the result is a queryable graph that grows continuously without user effort.

The target user is the person being out-produced by their own tools. In 2026, that's anyone working at agent-velocity: builders driving Claude Code who ship eight commits, four research docs, and a trait definition before lunch; researchers spawning parallel investigations faster than they can write up any single one. The volume of new context exceeds the rate at which any manual filing system can keep up.

The competitive frame isn't "biological memory" — nobody relies on that. It's the manual chain of capture-tool → graph-tool → agent-tool. Each link works individually; the seams are where velocity dies. Aurelius collapses the chain into a single pipeline with no seams.

03 — Three-Layer Architecture
01
Tier 1

Ingest

A Rust desktop shell sits on the user's Mac and captures raw accessibility events. High-volume, messy, low-signal on its own — the surveillance-camera footage of personal computing.

  • Tauri 2 native shell
  • macOS Accessibility API
  • EventKit (calendar)
  • NotificationCenter (banners)
02
Tier 2

Elicit

A Python sidecar feeds observation batches to Claude Sonnet (or Ollama for local) with an extraction prompt that produces atomic, typed, timestamped knowledge items linked back to source. Entity resolution merges duplicates; Leiden clustering surfaces themes.

  • Anthropic SDK (frontier path)
  • Ollama (local-first path)
  • Pydantic v2 schema validation
  • scikit-network Leiden clustering
03
Tier 3

Maintain & Query

Items decay, get contradicted, get superseded. The maintenance layer reconciles conflicts and prunes stale items. A fast-lookup cache serves natural-language queries against the graph — and an MCP server exposes Aurelius as a tool to Claude desktop and Claude Code.

  • SQLite shadow layer
  • NLI write-back validation
  • MCP server for Claude integration
  • BYOK abstraction (Claude / OpenAI / Ollama)

06 — Build Discipline
Validation Strategy

Gated Build

  • Three explicit gates with quantified success criteria
  • Gate 1: 500+ knowledge items, 20+ clusters, ER error < 15%
  • Gate 2: 50% relevance on elicited questions, 60% answer rate
  • Gate 3: 30% gap detection rate, staleness detection working
Architecture Posture

Spec-Driven Development

  • Spec first, code second — every feature defined before built
  • North star doc is upstream of architecture docs
  • Pin oracles before implementation — tests written first
  • No speculative features outside the spec
Privacy & Choice

Local-First, BYOK

  • Capture stays on the local machine via Tauri shell
  • Frontier path: Anthropic SDK with prompt caching
  • Local path: Ollama (Qwen3 8B default), zero API cost
  • Model-free tier works without any LLM dependency
