01 — Project Overview

DJI Cloud Simulator is a single-file, zero-build MQTT publisher that emits the exact wire-format messages a real DJI Dock 2 + Matrice 30T setup would emit — OSD telemetry, HMS alarms, mission progress events, device lifecycle transitions — against any MQTT broker. It exists because writing integrations against the DJI Cloud API without physical hardware is otherwise impossible: developers either buy a $30K+ dock or guess at the message format from incomplete docs.

The simulator is spec-accurate, not "close enough." Topic structure, QoS levels, payload schemas, nested status/progress/ext fields, and timing intervals all match the live DJI Cloud API. Adapter code that handles the simulator handles a real drone — verified against production deployments at Panoptris.

Open-sourced by Panoptris under MIT. Run with one command: npx github:YuYongJu/dji-cloud-simulator. No clone, no install, no config file required for the default patrol scenario.

02 — Built-In Scenarios

Each scenario exercises a different surface of the DJI Cloud API contract. Run with --scenario <name>.

Default

patrol

npx dji-cloud-simulator

Continuous patrol loop. OSD every 2s, dock telemetry every 10s, HMS alarms every 40s. Runs until Ctrl+C. The smoke test for any DJI Cloud adapter.

Lifecycle

mission

--scenario mission

Full mission flight: takeoff, four waypoints, landing. Publishes flighttask_progress events with nested status, progress, and ext fields matching the live spec.

Alarming

hms-alarm

--scenario hms-alarm

Five HMS alarm bursts with escalating cell imbalance. Compass interference fires first; battery imbalance joins once voltage differential exceeds the 200 mV threshold — matching real failure progression.

Edge Case

offline

--scenario offline

Device comes online, sends one OSD frame, then disconnects. Tests how your adapter handles device lifecycle transitions and stale-connection detection.

05 — Why It Exists
For Developers

Hardware-Free Iteration

  • Build DJI Cloud adapters without a $30K+ dock
  • Reproduce HMS alarm sequences for adapter unit tests
  • Single command via npx — no clone, no install
  • JSON config for custom waypoints, intervals, alarm codes
For Panoptris

CI Without Hardware

  • Drone integration tests run in GitHub Actions
  • Regression coverage for the MQTT adapter pipeline
  • Mission lifecycle tests don't need a physical dock
  • Open-sourcing benefits the broader DJI dev community
Engineering Choices

Disciplined Minimalism

  • Single dependency (mqtt.js) — small attack surface
  • Native node:test runner — no Vitest/Jest overhead
  • ESLint flat config v9 — LTS stability
  • Allowlist publish via files field — nothing leaks
Supply Chain

Modern Publish Posture

  • npm trusted publishing (OIDC) — no long-lived tokens
  • npm provenance attestation on every release
  • CI matrix across Node 18, 20, 22 LTS
  • Pre-publish dry-run validation in CI
